## 2024-05-24 - [Avoid `.toLowerCase()` and Array Methods inside Filter Loops over Large Payloads]
**Learning:** When dealing with large payloads (e.g. thousands of courses from a CDN), calling string transformation methods like `.toLowerCase()` inside a `.filter` or `.map` block inside a React `useMemo` leads to redundant string creations that severely degrade rendering performance and cause UI jank. Additionally, performing an `O(N)` lookup like `.some()` or `.find()` inside the iteration loop causes an `O(N*M)` execution time complexity.
**Action:** Always pre-compute transformed strings (like lowercase search queries) outside the loop before the `.filter()` function block. Convert nested array lookups to `O(1)` operations by precomputing `Set` objects before executing iterations over large arrays, particularly inside `useMemo` or component render blocks.
