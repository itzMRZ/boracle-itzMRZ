## 2024-05-24 - Frontend Chained Array Operations and String Transformations Bottleneck
**Learning:** Found a specific bottleneck in `src/app/(dashboard)/dashboard/preprereg/page.jsx` where large arrays of course data (~thousands of items) were being processed through multiple, chained `.filter()` methods. In each filter method, string transformations like `.toLowerCase()` were being computed repeatedly for every single item on static search terms. Additionally, an O(N) `.some()` check was being done inside the filter pass instead of pre-computing a `Set` for O(1) lookups. This caused significant UI jank as the runtime was forced to iterate the array multiple times, perform redundant memory allocations, and execute O(N*M) nested lookups during filtering.
**Action:** When working with large data sets in the frontend, always combine multiple filtering conditions into a single pass. Compute static string transformations (like search queries) outside of the loop so they are only calculated once. Pre-compute O(1) lookup tables (`Set` or `Map`) for existence checks against secondary arrays before iterating the main data array.
