## 2025-01-20 - Optimize Filtering and Array Lookups within Rendering Maps
**Learning:** The course list rendering in `src/app/(dashboard)/dashboard/preprereg/page.jsx` contains massive lists (hundreds of sections). Using `.find` on the `selectedCourses` array within the large `displayedCourses.map()` loop introduced a specific, severe O(N*M) algorithmic complexity that caused significant UI jank when scrolling or filtering the pre-registration table.
**Action:** Always utilize a `useMemo` to construct a `Set` out of the selected items array (e.g., `const selectedIds = new Set(selected.map(...))`) when you need to perform existence checks (`.has()`) inside an iteration over a large list like `displayedCourses`. This enables O(1) checks and maintains linear rendering performance even as the course catalog scales.
