## 2025-01-20 - Hoisting redundant calculations out of large array filters
**Learning:** In Next.js/React applications handling large sets of data directly on the client (like course catalogs), running string transformations such as `.toLowerCase()` repeatedly on the same search term inside `.filter()` loops creates significant overhead. In `preprereg/page.jsx`, filtering over ~1000 courses caused the application to evaluate `.toLowerCase()` multiple times per item (resulting in ~3000 redundant operations per keystroke search), creating an O(N) bottleneck that contributes to UI jank.
**Action:** Compute static transformations outside the `.filter` loop and assign them to constants. The same applies for `.some()` or `.find()` evaluations nested inside a filter map. Precompute the transformed states (e.g. mapping an array to a lowercase version) prior to the loop.
