## 2025-04-06 - Replacing O(N) Array methods with O(1) Set in React Render loops
**Learning:** In `src/app/(dashboard)/dashboard/preprereg/page.jsx`, rendering a list of up to 50 courses per page triggered `selectedCourses.find(...)` for every single row to check if the course was selected. While the selected course count was bounded (limit 25 credits, typically 8-9 courses), this structure could lead to noticeable jank when combined with complex layouts, and it's fundamentally an O(N*M) time complexity pattern. Similarly, the `.some()` function was used inside the filter logic.
**Action:** Instead of inline Array prototype methods (`.find()` or `.some()`), always pre-compute a `Set` or `Map` lookup using `useMemo` when rendering or mapping over an array to achieve O(1) evaluation within the iteration, particularly to protect the React render cycle from O(N^2) scaling bottlenecks.
